제공된 기획서와 기술 아키텍처를 바탕으로, 'Stock-Dakku: Active Trader Edition' 개발 마스터 플랜을 4단계로 나누어 제안합니다. 이 플랜은 데이터 지연 최소화와 실전 매매 효율성에 초점을 맞추고 있습니다.

--------------------------------------------------------------------------------
[Phase 1] 1-2주 차: 데이터 코어 및 인프라 구축 (Data Core)
프로젝트의 기반이 되는 데이터 파이프라인과 실시간 처리 환경을 설정하는 단계입니다.
• 백엔드 환경 설정: 비동기 처리에 유리한 Node.js를 기반으로 서버 아키텍처를 설계하고, 실시간 시세 데이터 처리를 위한 Redis 인-메모리 데이터베이스를 구축합니다.
• KIS API 연동: 한국투자증권(KIS) API를 활용해 웹소켓(Websocket) 커넥터를 개발합니다. 이를 통해 장중 투자자별 매매동향과 프로그램 매매 데이터를 실시간으로 수집합니다.
• 데이터 분기 처리:
    ◦ 핵심 종목: 웹소켓을 통한 실시간 푸시 시스템 구축.
    ◦ 일반 종목: API 호출 제한을 준수하기 위한 주기적 폴링(Polling) 및 캐싱 전략 구현.
[Phase 2] 3-4주 차: 차트 엔진 및 UI 구현 (Chart Engine)
수집된 데이터를 시각화하고 사용자 경험(UX)을 극대화하는 단계입니다.
• 차트 프레임워크 통합: React Native 환경에 금융 전용 라이브러리인 Lightweight Charts를 통합합니다.
• 복합 차트 구현: 가격 캔들 위에 외국인/기관 잠정 매수 추정치와 프로그램 순매수 데이터를 오버레이(Overlay)하여 실시간 수급 레이더 기능을 완성합니다.
• 디자인 시스템 적용: 다크 네이비(#0b1219) 색상 기반의 대시보드 UI를 적용하고, 상승(Red)과 하락(Blue)이 명확히 대비되도록 시각화합니다.
[Phase 3] 5-6주 차: 핵심 분석 알고리즘 및 AI 통합 (Advanced Logic)
기획서의 킬러 기능을 완성하고 데이터 정합성을 검증하는 단계입니다.
• 호가창 파워 분석 엔진: 매수/매도 잔량 비율을 계산하고, 체결강도 120% 이상 및 순매수 체결량 급증 시 매수 신호를 발생시키는 알고리즘을 코딩합니다.
• 세력 평단가 추적기: 최근 3일, 5일, 20일간의 데이터를 분석하여 세력의 추정 평균 단가를 차트에 표시하는 로직을 구현합니다.
• Sentiment AI 연동: 실시간 뉴스를 분석하여 호재/악재를 판별하고, 주가 영향도를 1~10점으로 점수화(Impact Score)하는 기능을 통합합니다.
• 알파 테스트: 실제 HTS와 데이터를 비교하여 수치의 정확성을 검증하는 정합성 테스트를 수행합니다.
[Phase 4] Launch 및 수익화 시스템 (Monetization & Launch)
배포와 유료 서비스 운영을 위한 시스템을 마무리합니다.
• 구독 시스템 구축: 무료(Basic)와 유료(Pro) 사용자의 데이터 접근 권한을 분리합니다. Pro 사용자에게는 실시간 수급, 호가 데이터, 세력 평단가 차트를 제공합니다.
• 시그널 푸시 서비스: 특정 수급 변곡점(예: 외국인 대량 순매수 돌파) 발생 시 실시간으로 알림을 보내는 건당 과금 모델 연동을 준비합니다.
• 마케팅 및 출시: 앱스토어 배포와 함께 인플루언서 제휴 등 초기 사용자 확보를 위한 활동을 전개합니다.

기술적 통찰 (Technical Insight): 이 플랜의 핵심은 데이터의 무결성과 저지연성입니다. 소스에 따르면, 주식 시세와 같은 초단기 데이터는 DB가 아닌 Redis에서 직접 처리하도록 설계되어 있어, 서버 부하를 줄이면서도 사용자에게는 HTS 급의 빠른 속도를 제공할 수 있습니다. 또한, AI 감성 분석 모델의 경우 외부 API나 라이브러리를 선정하여 Impact Score 산출 로직에 정교하게 반영하는 과정이 필요합니다 (해당 세부 모델명은 소스에 명시되지 않았으므로 별도 선정이 필요할 수 있습니다).
